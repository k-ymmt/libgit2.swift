# Git2 Swift Wrapper — v0.5c-i Repository Init Slice

**Status:** Draft
**Date:** 2026-04-21
**Scope:** First slice of the v0.5c "git CLI porcelain catch-up" line. Adds
`Repository.create(at:bare:initialBranch:)` as a thin wrapper over
`git_repository_init_ext`. Builds on v0.2
(`docs/superpowers/specs/2026-04-20-git2-swift-wrapper-foundation-design.md`)
through v0.5b-ii
(`docs/superpowers/specs/2026-04-21-git2-v0.5b-ii-network-push-design.md`).

## 1. Purpose

Every prior slice assumed a repository already existed on disk. Tests
work around this by calling `git_repository_init` directly through
`@testable import Cgit2` in three places:

1. **Two test-support helpers** — `Tests/Git2Tests/Support/Helpers.swift`'s
   `initRepo(at:)` / `initBareRepo(at:)` (consumed by ~30 tests that call
   the helpers instead of inlining).
2. **Two test-support fixtures** — `LocalRemoteFixture.swift` (bare
   upstream + non-bare downstream) and `TestFixture.swift` (linear /
   merge history builders).
3. **Fifteen inline call sites** across 11 test files, each opening a
   fresh empty repo the same three-line way:
   `RemoteGitHubIntegrationTests.swift` × 3,
   `ReferenceSequenceTests.swift`, `ReferenceDeleteTests.swift`,
   `Concurrency/WriteConcurrencyTests.swift`,
   `Repository/RepositoryOpenTests.swift`,
   `Repository/RepositoryHeadTests.swift`,
   `Repository/RepositoryCommitsTests.swift`,
   `Repository/RepositoryTagsTests.swift`,
   `Repository/RepositoryTreesTests.swift`,
   `Repository/RepositoryBranchesTests.swift`,
   `Repository/RepositoryBlobsTests.swift` × 3.

Library consumers also have no public way to create a repository at
all.

v0.5c-i closes the first of the two gaps listed under
`TODO.md` → "Missing git CLI porcelain":

- **Public initializer.** `Repository.create(at:bare:initialBranch:)`
  mirrors the existing `Repository.open(at:)` shape (static method,
  `throws(GitError)`, same bootstrap precondition).
- **Test-layer migration.** `initRepo` / `initBareRepo` in
  `Tests/Git2Tests/Support/Helpers.swift` switch their internals from
  `git_repository_init` to `Repository.create`. The two Support
  fixtures (`LocalRemoteFixture`, `TestFixture`) and all 15 inline
  call sites switch to the public API as well. Function signatures of
  the `Support/` helpers / fixtures remain unchanged so their ~30
  call-site tests are untouched.

v0.5c-i is **additive**: every v0.2–v0.5b-ii public API keeps the same
signature and behavior. The wrapper's thin-over-libgit2 policy is
preserved — initial-branch resolution is delegated to libgit2's own
`init.defaultBranch` / `"master"` fallback logic.

The second gap (`Repository.pull` porcelain) is the subject of a
separate upcoming slice (tentatively `v0.5c-ii`).

## 2. Target Platforms

Unchanged from v0.5b-ii: Apple platforms only, macOS 13+, iOS 16+,
Swift 6.2. No SSH, no SHA-256. `create` is a pure local-disk operation
and adds no new platform requirements.

### 3.2 Explicitly not included in v0.5c-i

All tracked in TODO.md under "Deferred from v0.5c-i (repository init slice)".

- **`Repository.InitOptions` value type.** Options struct becomes
  worth the extra surface only when multiple advanced knobs land
  together. Current slice keeps the two scalar arguments.
- **`failIfExists: Bool` (`GIT_REPOSITORY_INIT_NO_REINIT`).** libgit2's
  default reinitialize behavior (re-apply config, preserve refs /
  objects / HEAD) is idempotent and safe; consumers can pre-check with
  `FileManager.default.fileExists(atPath:)` if they need the "fail
  already-exists" semantics.
- **Shared mode (`GIT_REPOSITORY_INIT_SHARED_UMASK` / `_GROUP` / `_ALL`).**
  Permission-bit control for multi-user server hosting. Adds as
  `SharedMode` enum when a concrete use case appears.
- **External template path (`GIT_REPOSITORY_INIT_EXTERNAL_TEMPLATE` +
  `template_path`).** Custom init templates.
- **`origin_url`.** Auto-create an `origin` remote pointing at the
  supplied URL during init. Trivially composed today via `create` +
  `createRemote(named:url:)`.
- **`workdir_path` (non-natural working tree).** Gitlink-file-based
  alternate worktree. Lands with (or adjacent to) a future
  `worktree` slice.
- **`description` file.** Repository description string used by GitWeb
  / cgit. Low-priority cosmetic.
- **`RELATIVE_GITLINK` flag.** Pairs with `workdir_path`; defer together.
- **`refdb_type`.** libgit2 supports `files` and (newer) `reftable`.
  `files` is the default and only option the XCFramework currently
  exposes without rebuilds.
- **SHA-256 repositories (`oid_type`).** Gated on `EXPERIMENTAL_SHA256=ON`
  at XCFramework build time — tracked under the existing TODO item.
- **`Repository.pull` porcelain.** The second "Missing git CLI
  porcelain" TODO item, deferred to slice v0.5c-ii so each porcelain
  ships with its own spec / plan / review cycle.

### 3.3 Non-goals (unchanged)

- No shipped CLI.
- No 1:1 exposure of every libgit2 function.
- No Linux / Windows / Android.

## 4. Architectural Continuity

Every v0.2–v0.5b-ii decision carries forward unchanged:

- **Thin-wrapper-over-libgit2.** `create` delegates initial-branch
  resolution to libgit2 (reads `init.defaultBranch`, falls back to
  `"master"`) rather than pinning `"main"` in the Swift layer. Same
  spirit as the v0.5a-ii "don't normalize `ontoName`/`origHeadName`
  asymmetry" decision and the v0.5b-ii "don't make rejection messages
  machine-readable" decision.
- **Bootstrap precondition.** `Git.bootstrap()` must have been called —
  same gate as `open`. Forgetting trips a `preconditionFailure` via
  `GitRuntime.shared.requireBootstrapped()`.
- **URL → path bridging.** `URL.withUnsafeFileSystemRepresentation` →
  `GIT_EINVALIDSPEC` on nil path, matching `open`'s sentinel pattern.
  The existing minor-polish TODO item ("Repository.open sentinel when
  withUnsafeFileSystemRepresentation yields nil — preconditionFailure
  would be clearer") applies symmetrically to `create`; both resolve
  together if and when that sweep happens.
- **No Repository lock.** `create` is a static method that produces a
  fresh `Repository` handle. Filesystem-level synchronization is
  libgit2's responsibility on the write path.
- **Error handling.** libgit2 return codes flow through the existing
  `check(rc)` helper into `GitError`. §9.2's "error-code specificity"
  policy applies to the test assertions (§7).

## 5. Public API

One method added to `Repository`.

### 5.1 `Repository.create(at:bare:initialBranch:)`

```swift
extension Repository {
    /// Creates (or reinitializes) a Git repository at `url`.
    ///
    /// Thin wrapper over `git_repository_init_ext` with
    /// `GIT_REPOSITORY_INIT_MKPATH` pre-set, so missing intermediate
    /// directories under `url` are created automatically.
    ///
    /// Reinitializing an existing repository is **safe and idempotent**:
    /// libgit2 re-applies config but preserves the existing object
    /// database, refs, and HEAD. HEAD is re-targeted only when
    /// `initialBranch` is explicitly non-`nil` on the reinit call.
    ///
    /// - Parameters:
    ///   - url: Working-tree directory for non-bare repositories; the
    ///     repository root directory for bare repositories.
    ///   - bare: `true` to create a bare repository (no working tree).
    ///     Default `false`.
    ///   - initialBranch: Name of the initial branch written into HEAD
    ///     (without the `refs/heads/` prefix — libgit2 adds it). `nil`
    ///     delegates to libgit2, which reads `init.defaultBranch` from
    ///     the user's gitconfig and falls back to `"master"`. Default
    ///     `nil`.
    /// - Returns: An opened ``Repository`` handle for the new
    ///   (or reinitialized) repository.
    /// - Throws: ``GitError`` — ``GitError/Code/invalidSpec`` when `url`
    ///   yields no file-system representation; libgit2 code propagation
    ///   otherwise (``GitError/Code/os`` / `filesystem` for permission
    ///   or I/O failures, invalid-reference codes for malformed
    ///   `initialBranch` values).
    /// - Precondition: ``Git/bootstrap()`` has been called.
    public static func create(
        at url: URL,
        bare: Bool = false,
        initialBranch: String? = nil
    ) throws(GitError) -> Repository
}
```

### 5.2 Reinit semantics (documented behavior)

Empirically verified against libgit2 1.9.x
(`src/libgit2/repository.c` → `git_repository_init_ext`):

| State before call | `initialBranch` on reinit | Effect |
|---|---|---|
| Path does not exist | any | fresh `.git/` layout, HEAD symbolic-ref to `refs/heads/<branch>` (or libgit2 default if `nil`) |
| Fresh non-repo directory | any | as above |
| Existing repository | `nil` | config re-applied; refs / objects / HEAD untouched |
| Existing repository | non-`nil` | config re-applied; HEAD symbolic-ref retargeted; refs / objects untouched |
| Existing repository with mismatched `bare` | any | libgit2 may error (`GIT_EEXISTS` / `.repository`); observable, not swallowed |

This matches `git init` CLI's "Reinitialized existing Git repository"
behavior.

## 6. Implementation

### 6.1 File placement

Create `Sources/Git2/Repository/Repository+Create.swift` alongside the
other `Repository+*.swift` topical files. Keeps `Repository.swift`
focused on the core handle type and `open`.

### 6.2 Implementation sketch

```swift
import Cgit2
import Foundation

extension Repository {
    public static func create(
        at url: URL,
        bare: Bool = false,
        initialBranch: String? = nil
    ) throws(GitError) -> Repository {
        GitRuntime.shared.requireBootstrapped()

        var opts = git_repository_init_options()
        let initRC = git_repository_init_options_init(
            &opts,
            UInt32(GIT_REPOSITORY_INIT_OPTIONS_VERSION)
        )
        precondition(
            initRC == 0,
            "git_repository_init_options_init failed (libgit2 version mismatch?)"
        )

        var flags = UInt32(GIT_REPOSITORY_INIT_MKPATH.rawValue)
        if bare {
            flags |= UInt32(GIT_REPOSITORY_INIT_BARE.rawValue)
        }
        opts.flags = flags

        var raw: OpaquePointer?
        let result: Int32 = url.withUnsafeFileSystemRepresentation { path -> Int32 in
            guard let path else { return GIT_EINVALIDSPEC.rawValue }
            if let branch = initialBranch {
                return branch.withCString { branchPtr -> Int32 in
                    opts.initial_head = branchPtr
                    return git_repository_init_ext(&raw, path, &opts)
                }
            } else {
                opts.initial_head = nil
                return git_repository_init_ext(&raw, path, &opts)
            }
        }
        try check(result)
        return Repository(handle: raw!)
    }
}
```

The two-branch inlining on `initialBranch` is intentional. The shared
`withOptionalCString` helper is `throws(GitError)` on the body, which
composes cleanly at the top level of a throwing function (see
`Rebase.commit`) but introduces typed-throws-vs-untyped-rethrows
boilerplate when nested inside `URL.withUnsafeFileSystemRepresentation`
(untyped `rethrows`). Since neither of the closures here actually
throws, keeping both bodies non-throwing matches `Repository.open`'s
shape and avoids the bridge.

### 6.3 Helper / runtime reuse

- `GitRuntime.shared.requireBootstrapped()` — same bootstrap gate as
  `open`.
- `check(_:)` — existing `GitError` conversion used everywhere.
- `String.withCString` / `URL.withUnsafeFileSystemRepresentation` —
  Foundation primitives, same pattern as `open`.

No new internal helpers are introduced. The shared
`withOptionalCString` is deliberately not used here (see §6.2).

### 6.4 Lifetime notes

`opts.initial_head` points at the CString produced inside the
`withOptionalCString` closure. `git_repository_init_ext` is called
synchronously inside that closure, so the pointer outlives the libgit2
call by construction. The `opts` struct is stack-local and dropped on
return.

### 6.5 Locking

`create` is a static factory — no `Repository` handle exists yet, so
no instance lock applies. Filesystem-level synchronization of
concurrent `create` calls targeting the same path is libgit2's / the
OS's responsibility, consistent with how `open` treats concurrent
opens.

### 6.6 Why `create` rather than `init`

`Repository.init(at:)` would collide with Swift's initializer syntax
(and require backticks at every call site). Project convention uses
verb-named static methods for constructing handles (`open`, plus
instance-level `createRemote`, `createBranch`, `startRebase`).
`Repository.create(at:)` parallels the existing `Repository.open(at:)`
naturally.

### 7.3 Out of scope for this suite

- **`GIT_EINVALIDSPEC` Swift-layer sentinel (`guard let path else
  { return GIT_EINVALIDSPEC.rawValue }`).** Empirically verified
  during v0.5c-i implementation that `URL.withUnsafeFileSystemRepresentation`
  never returns `nil` on Apple platforms for the URL shapes we
  surveyed (`http://`, `ftp://`, `mailto:`, malformed strings — all
  produce a non-nil C-string). The sentinel fallback is therefore
  dead code on supported platforms. It is retained for defensive
  parity with `Repository.open(at:)` (which carries the identical
  sentinel and is also not exercised by `RepositoryOpenTests`). No
  Apple-platform test can realistically drive this branch; the TODO
  minor-polish item ("Repository.open sentinel when
  `withUnsafeFileSystemRepresentation` yields nil — `preconditionFailure`
  would be clearer") applies symmetrically to `create` and is the
  right place to resolve this.
- **`import Cgit2` removal sweep.** `Helpers.swift` and many test
  files still need `import Cgit2` for unrelated reasons
  (`git_filemode_t`, `git_index_*` with stage encoding, etc.). Full
  `@testable import Cgit2` cleanup remains the v0.4a / v0.4b-i
  deferred item.
- **Bootstrap precondition test.** `Git.bootstrap()` is shared
  initialization for the entire test target; a "calling before
  bootstrap trips precondition" test would require isolating the
  shared runtime, which no other suite does. `RepositoryOpenTests`
  does not assert this either.
- **TSan / concurrency test.** `create` produces no handle-bound state
  before returning; concurrent invocations targeting disjoint paths
  cannot race on Swift-layer state, and same-path races reduce to the
  idempotent-reinit case already covered by test #6. No dedicated
  concurrency suite is added.

## 8. Errors

Observed (or expected) `GitError` codes:

| Situation | `code` | `class` | Source |
|---|---|---|---|
| `url` produces no file-system representation | `.invalidSpec` | — | Swift-layer sentinel, matches `open` |
| Parent path does not exist and cannot be created (permission) | `.os` / `.filesystem` | — | libgit2 propagation |
| `initialBranch` cannot form a valid `refs/heads/<name>` | `.invalidSpec` | `.reference` | libgit2 validation (verify empirically in test #9) |
| `bare`-vs-existing-non-bare mismatch | `.exists` | `.repository` | libgit2 propagation (empirical) |
| Race: another process creates `.git/` concurrently | `.exists` | `.filesystem` | libgit2 propagation (rare) |

Reinitializing an existing repository with the same `bare` flag does
**not** throw (idempotent; see §5.2 and test #6).

## 11. Forward Pointers

- **`Repository.pull` porcelain (`v0.5c-ii`).** Composition of
  `fetch(remoteNamed:)` → `annotatedCommit(for:)` → `mergeAnalysis` →
  `merge(branchNamed:)` dispatch. Open design questions: rebase-style
  vs merge-style dispatch flag, whether to surface the analysis result
  to the caller, conflict surface, and auto-commit on the `.normal`
  merge path (which is itself blocked on the v0.5a-i "auto-commit
  porcelain" deferred item).
- **`Repository.InitOptions` value type.** Introduced when two or more
  of the §3.2 deferred knobs (`sharedMode`, `templatePath`,
  `originURL`, `description`, `failIfExists`) land together. The
  scalar-argument `create(at:bare:initialBranch:)` stays as a
  convenience overload.
- **Worktree slice.** Brings `workdir_path` + `RELATIVE_GITLINK` into
  scope alongside `git_worktree_*` APIs.
- **Full `@testable import Cgit2` sweep.** Separate cosmetic pass; not
  gated on any porcelain item.
