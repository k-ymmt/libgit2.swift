# Git2 Swift Wrapper — v0.6 Status Surface

**Status:** Draft
**Date:** 2026-04-24
**Scope:** Single-slice wrapping of the full libgit2 `status.h` surface.
Adds a snapshot API (`Repository.statusEntries(options:)`), a live handle
class (`StatusList`), a single-path query (`Repository.status(forPath:)`),
and an ignore predicate (`Repository.shouldIgnore(path:)`), together with
the backing value types `StatusEntry`, `StatusFlags`, and `StatusOptions`.
Builds on v0.3 (read extensions; `Diff` / `DiffDelta`), v0.4b-i (`Index`),
v0.5a-i (merge / cherry-pick; supplies `TestFixture+Merge` for the
conflicted-entry test), and v0.5c-ii (pull porcelain — most recent slice).

## 1. Purpose

`TODO.md` lists `status` under "Missing git CLI porcelain" implicitly —
the section names `pull` (landed in v0.5c-ii) and `create` (v0.5c-i) as
the first two gaps filled after the v0.5b-ii audit, with `status` as the
next natural surface. The porcelain is fundamental: every other porcelain
that inspects the working tree (pull, merge, checkout) would benefit from
a user-facing way to observe what changed between HEAD, the index, and
the working tree.

libgit2's `status.h` is a well-bounded C surface:

| libgit2 entry point | Purpose |
|---|---|
| `git_status_list_new` + `_entrycount` + `_byindex` + `_free` | Snapshot list of per-file status |
| `git_status_foreach` / `_foreach_ext` | Callback-form iteration |
| `git_status_file` | Single exact-path lookup |
| `git_status_should_ignore` | Ignore-rule predicate |
| `git_status_options` + `git_status_opt_t` + `git_status_show_t` | Options (show / flags / pathspec / baseline / rename_threshold) |
| `git_status_entry` | Per-file result (status flags + `head_to_index` / `index_to_workdir` `git_diff_delta *`) |

v0.6 surfaces the list / single-path / should-ignore paths directly and
defers the callback-form iteration (`foreach_ext`) to the unified
callback-bridging slice discussed in v0.4b-ii / v0.5a-i / v0.5a-ii
follow-ups. The snapshot-and-handle coverage is the natural Swift shape
for the majority of use cases (UI "what changed?" displays, CI pre-commit
checks, porcelain composition).

v0.6 is **additive**: every v0.2–v0.5c-ii public API keeps the same
signature and behavior. No refactor of existing types is required;
`DiffDelta` is reused unchanged.

## 2. Target Platforms

Unchanged from v0.5c-ii: Apple platforms only, macOS 13+, iOS 16+,
Swift 6.2. No SSH, no SHA-256. Status reuses only already-linked
libgit2 symbols; no XCFramework rebuild.

### 3.2 Explicitly not included in v0.6

Tracked in `TODO.md` under "Deferred from v0.6 (status slice)":

- **`forEachStatus(options:) { ... }` callback-form API**
  (`git_status_foreach` / `_foreach_ext`). v0.6 covers both snapshot and
  handle shapes; the callback path adds only "early termination" as
  incremental value. Bridging Swift closures to the C callback has the
  same context-pointer / cancellation-propagation shape as the deferred
  checkout / merge / rebase callbacks (v0.4b-ii / v0.5a-i / v0.5a-ii
  TODOs). Land as one unified callback-bridging slice rather than
  one-per-surface.
- **`StatusOptions` presets.** `.porcelain`, `.gitStatusCliDefaults`, etc.
  The `StatusOptions(show:flags:pathspec:baseline:renameThreshold:)`
  initializer with a default `.defaults` flag set already hits the
  common case. Presets land when a concrete repeat pattern appears at
  a call site.
- **`StatusList` `Sequence` conformance.** v0.6 exposes `count` +
  `subscript(Int)` only (matches `Diff`). Add when a caller wants
  `for entry in list` without the explicit index loop.
- **Submodule-aware status coverage.** v0.6's test surface exercises
  only `excludeSubmodules` (flag behavior, via synthetic `.gitignore`
  that hides the would-be submodule directory). Comprehensive
  submodule status (`SUBMODULE_NEW`, dirty submodule, etc.) lands with
  a dedicated submodule slice once the wrapper exposes submodule
  read/write APIs. Introducing a real submodule in the fixture today
  requires `@testable` reach-in to `Cgit2` (no public submodule API
  yet), which v0.6 avoids to keep the fixture layer closer to pure
  public API.
- **`IndexStatus` / `WorkdirStatus` split enums.** A shape where the
  per-file result carries two distinct enums (`indexStatus:
  IndexStatus?`, `workdirStatus: WorkdirStatus?`) plus flat
  `isConflicted` / `isIgnored` booleans, instead of a single
  `OptionSet`. v0.6's `StatusFlags` `OptionSet` with convenience
  computed properties (`hasIndexChanges`, `hasWorkdirChanges`,
  `isConflicted`, `isIgnored`, `isCurrent`) covers the common
  discrimination needs without round-tripping libgit2's bitmask into
  two enums. Revisit if call sites repeatedly spelunk the flags in
  ways the convenience properties don't capture.
- **Independent path exposure on `StatusEntry`** outside the
  `DiffDelta?` pair. `git_status_entry` itself has no `path` field —
  libgit2 expects callers to read from the embedded `git_diff_file *`.
  v0.6 exposes `path: String` computed once at entry construction from
  `headToIndex?.newFile.path ?? indexToWorkdir?.newFile.path ??
  indexToWorkdir?.oldFile.path`. The fallback chain is stable because
  a non-`.current` entry always has at least one side non-`NULL`. If a
  future API consumer needs the raw byte path (for non-UTF-8 filenames
  on HFS+ / case-insensitive filesystems), add a sibling
  `pathBytes: [UInt8]` — deferred because no consumer has appeared.
- **SHA-256 repository status.** Gated on the repository-wide
  `EXPERIMENTAL_SHA256=ON` TODO.
- **`StatusList` consuming `close()`.** Explicit drop to replace
  `deinit`-triggered `git_status_list_free`. Same ergonomics question
  as every other handle class (`Diff`, `Rebase`, …); lands across all
  of them together when the pattern is needed.
- **Linked-worktree status.** Worktree slice hasn't landed; status in a
  linked worktree uses `git_repository_open` on the worktree path today
  (libgit2's worktree awareness is in `git_worktree.h`, a separate
  surface).
- **Rename detection in `status(forPath:)`.** libgit2 explicitly
  documents that `git_status_file` does *not* do rename detection —
  renames require the full-list scan path. v0.6 mirrors this
  limitation. A Swift-layer "run `statusEntries`, filter to `path`"
  helper is composable but adds a hidden O(n) cost under a single-path
  signature and is deferred until a call site requests it.
- **`StatusFlags.description` / `CustomStringConvertible`.** Debug
  pretty-printing. Additive; defer until a debugging need appears.
- **Error-code specificity tightening.** v0.6's error tests assert
  `.notFound` / `.ambiguous` / `.bareRepo` per the libgit2 contract.
  Cross-slice pattern tightening (class + code pairs, message pinning)
  waits for the horizontal pass already queued in v0.3 / v0.4a /
  v0.4b-i / v0.4b-ii / v0.5a-i / v0.5c-ii follow-ups.

### 3.3 Design discussion — decisions

The design was brainstormed through the following choice points (all
recorded here so the rationale is reachable from the spec alone):

- **Full `status.h` surface in this slice** (vs. porcelain minimum).
  Chosen because the four entry points share the same options value
  type; splitting would force redesign of options across sub-slices.
- **Four public entry points** (snapshot array, handle class,
  single-path, ignore predicate). `foreachExt` callback form deferred
  (see §3.2).
- **`StatusFlags` as a single `OptionSet`** (vs. split index/workdir
  enums). Keeps the libgit2 mapping 1:1 and avoids modeling concerns
  around flag combinations libgit2 permits but the split enums would
  reject. Convenience computed properties bridge the ergonomics gap.
- **`StatusOptions` as a value type with all five fields**
  (`show`, `flags`, `pathspec`, `baseline`, `renameThreshold`) and
  defaulted initializer parameters. No presets in v0.6.
- **One slice, not two.** Options surface is shared; splitting would
  double the CHANGELOG / spec / plan bookkeeping without helping review
  granularity.

## 4. Architectural Continuity

v0.6 inherits the established wrapper shape:

- **Thin-wrapper policy.** Swift types map 1:1 onto libgit2 C types.
  Pre-checks are absent; libgit2 owns validation. Exceptions documented
  where they appear (there are none in v0.6).
- **Typed throws.** All APIs that can fail throw `GitError`
  (`throws(GitError)`). `shouldIgnore(path:)` throws because
  `git_status_should_ignore` returns an error code, even though in
  practice it succeeds on any input.
- **Repository-scoped lock.** Every libgit2 call sits inside
  `repository.lock.withLock { ... }`. Handle classes that own
  libgit2 pointers (`StatusList`, like `Diff`) defer to the repository
  lock for subscript reads.
- **`@unchecked Sendable` handles.** `StatusList` follows `Diff`'s
  pattern: strong reference to the owning `Repository`, lock-gated
  subscript, `deinit`-triggered free.
- **`DiffDelta` reuse.** `StatusEntry.headToIndex` and
  `.indexToWorkdir` hold `DiffDelta?` values directly — no new delta
  type.
- **File layout.** New top-level `Sources/Git2/Status/` directory
  matching `Diff/`, `Walker/`, etc. New `Tests/Git2Tests/Status/`
  matching `Tests/Git2Tests/Diff/`, etc.

## 5. Public API

### 5.1 `Repository+Status.swift`

```swift
import Cgit2

extension Repository {
    /// Returns a snapshot of every file whose index/workdir status matches `options`.
    ///
    /// Implemented as `statusList(options:)` followed by an eager
    /// `(0..<count).map { self[$0] }` — the underlying `git_status_list`
    /// is freed before this method returns.
    ///
    /// - Throws: ``GitError``. Notably ``GitError/Code/bareRepo`` when the
    ///   repository is bare.
    public func statusEntries(
        options: StatusOptions = StatusOptions()
    ) throws(GitError) -> [StatusEntry]

    /// Opens a live handle over `git_status_list`. The caller holds the
    /// handle for as long as they want to inspect entries lazily.
    ///
    /// The entry set is frozen at creation time — subsequent working-tree
    /// mutations are not reflected until a fresh list is requested.
    ///
    /// - Throws: ``GitError``. Notably ``GitError/Code/bareRepo`` when the
    ///   repository is bare.
    public func statusList(
        options: StatusOptions = StatusOptions()
    ) throws(GitError) -> StatusList

    /// Status flags for an exact path, relative to the working directory.
    ///
    /// Does **not** perform rename detection — libgit2's `git_status_file`
    /// cannot. If the caller needs rename flags, use `statusEntries`
    /// with `renamesHeadToIndex` / `renamesIndexToWorkdir`.
    ///
    /// - Throws: ``GitError``.
    ///   - ``GitError/Code/notFound`` if no such path exists in HEAD, the
    ///     index, or the working tree.
    ///   - ``GitError/Code/ambiguous`` if `path` matches multiple files
    ///     or refers to a directory.
    ///   - ``GitError/Code/bareRepo`` on a bare repository.
    public func status(forPath path: String) throws(GitError) -> StatusFlags

    /// Whether gitignore rules would ignore `path`.
    ///
    /// This consults ignore rules only; tracked files may still return
    /// `true` because libgit2 reports the rules verdict, not the actual
    /// tracking state. See ``StatusFlags/ignored`` for the combined verdict.
    ///
    /// - Throws: ``GitError`` (in practice always returns successfully
    ///   — libgit2 returns `0` or `1` for any path).
    public func shouldIgnore(path: String) throws(GitError) -> Bool
}
```

### 5.2 `StatusList.swift`

```swift
import Cgit2

/// A live handle over `git_status_list`. Reads are serialized through
/// the owning repository's lock. The entry set is frozen at the moment
/// the list was created.
public final class StatusList: @unchecked Sendable {
    internal let handle: OpaquePointer
    public let repository: Repository

    internal init(handle: OpaquePointer, repository: Repository) {
        self.handle = handle
        self.repository = repository
    }

    deinit {
        git_status_list_free(handle)
    }

    /// Number of status entries in the list.
    public var count: Int {
        repository.lock.withLock { git_status_list_entrycount(handle) }
    }

    /// Returns the entry at `index` (0-based). Out-of-range access traps.
    public subscript(index: Int) -> StatusEntry {
        repository.lock.withLock {
            guard let raw = git_status_byindex(handle, index) else {
                preconditionFailure("StatusList subscript out of range: \(index)")
            }
            return StatusEntry(raw: raw.pointee)
        }
    }
}
```

### 5.3 `StatusEntry.swift`

```swift
import Cgit2

public struct StatusEntry: Sendable, Equatable {
    /// UTF-8 path relative to the working directory. Resolved at read time
    /// from whichever of `headToIndex.newFile` / `indexToWorkdir.newFile` /
    /// `indexToWorkdir.oldFile` is non-NULL, in that priority.
    public let path: String

    /// Combined status flags (a union of `INDEX_*`, `WT_*`, `IGNORED`,
    /// `CONFLICTED`). Empty for the unmodified-but-reported case,
    /// which only appears when `StatusOptions.Flags.includeUnmodified`
    /// is set.
    public let flags: StatusFlags

    /// HEAD→index delta, if any index-side flag is set.
    public let headToIndex: DiffDelta?

    /// Index→workdir delta, if any workdir-side flag is set.
    public let indexToWorkdir: DiffDelta?
}

extension StatusEntry {
    internal init(raw: git_status_entry) {
        let headToIndex = raw.head_to_index.map { DiffDelta(raw: $0.pointee) }
        let indexToWorkdir = raw.index_to_workdir.map { DiffDelta(raw: $0.pointee) }
        let resolvedPath =
            headToIndex?.newFile.path
            ?? indexToWorkdir?.newFile.path
            ?? indexToWorkdir?.oldFile.path
            ?? ""
        self.path = resolvedPath
        self.flags = StatusFlags(rawValue: UInt32(raw.status.rawValue))
        self.headToIndex = headToIndex
        self.indexToWorkdir = indexToWorkdir
    }
}
```

### 5.4 `StatusFlags.swift`

```swift
public struct StatusFlags: OptionSet, Sendable, Hashable {
    public let rawValue: UInt32
    public init(rawValue: UInt32) { self.rawValue = rawValue }

    public static let indexNew        = StatusFlags(rawValue: 1 << 0)
    public static let indexModified   = StatusFlags(rawValue: 1 << 1)
    public static let indexDeleted    = StatusFlags(rawValue: 1 << 2)
    public static let indexRenamed    = StatusFlags(rawValue: 1 << 3)
    public static let indexTypeChange = StatusFlags(rawValue: 1 << 4)

    public static let wtNew           = StatusFlags(rawValue: 1 << 7)
    public static let wtModified      = StatusFlags(rawValue: 1 << 8)
    public static let wtDeleted       = StatusFlags(rawValue: 1 << 9)
    public static let wtTypeChange    = StatusFlags(rawValue: 1 << 10)
    public static let wtRenamed       = StatusFlags(rawValue: 1 << 11)
    public static let wtUnreadable    = StatusFlags(rawValue: 1 << 12)

    public static let ignored         = StatusFlags(rawValue: 1 << 14)
    public static let conflicted      = StatusFlags(rawValue: 1 << 15)

    /// Union of all INDEX_* flags.
    public static let indexChanges: StatusFlags =
        [.indexNew, .indexModified, .indexDeleted, .indexRenamed, .indexTypeChange]

    /// Union of all WT_* flags.
    public static let workdirChanges: StatusFlags =
        [.wtNew, .wtModified, .wtDeleted, .wtTypeChange, .wtRenamed, .wtUnreadable]

    public var hasIndexChanges: Bool   { !isDisjoint(with: .indexChanges) }
    public var hasWorkdirChanges: Bool { !isDisjoint(with: .workdirChanges) }
    public var isConflicted: Bool      { contains(.conflicted) }
    public var isIgnored: Bool         { contains(.ignored) }
    /// Matches `GIT_STATUS_CURRENT` (no bits set).
    public var isCurrent: Bool         { rawValue == 0 }
}
```

        public static let disablePathspecMatch         = Flags(rawValue: 1 << 5)

## 6. Implementation

### 6.1 Options bridging (`Internal/StatusOptionsBridge.swift`)

The bridge converts `StatusOptions` to a temporary `git_status_options`
struct whose lifetime is bounded to a single libgit2 call:

```swift
internal func withBridgedStatusOptions<R>(
    _ options: StatusOptions,
    _ body: (UnsafePointer<git_status_options>) throws(GitError) -> R
) throws(GitError) -> R {
    var raw = git_status_options()
    // Version init — never fails per libgit2 contract; assert ok.
    let initRC = git_status_options_init(&raw, UInt32(GIT_STATUS_OPTIONS_VERSION))
    precondition(initRC == 0, "git_status_options_init failed")

    raw.show = options.show.rawValue
    raw.flags = options.flags.rawValue
    raw.rename_threshold = options.renameThreshold

    return try options.pathspec.withCStringArray { cstrings throws(GitError) in
        raw.pathspec.strings = cstrings.baseAddress
        raw.pathspec.count = cstrings.count

        // Hold Tree strong reference across the call.
        return try withExtendedLifetime(options.baseline) {
            raw.baseline = options.baseline?.handle
            return try withUnsafePointer(to: &raw) { ptr throws(GitError) in
                try body(ptr)
            }
        }
    }
}
```

- `[String].withCStringArray` is a new internal helper (or reuse any
  existing one from `Refspec` / pathspec-consuming call sites — the
  grep during implementation picks the existing helper if it matches
  signature).
- `git_status_options_init` is asserted with `precondition` per
  wrapper convention for "initializer that never fails in practice".

### 6.2 `Repository.statusList(options:)`

```swift
public func statusList(
    options: StatusOptions = StatusOptions()
) throws(GitError) -> StatusList {
    try lock.withLock { () throws(GitError) -> StatusList in
        try withBridgedStatusOptions(options) { optsPtr throws(GitError) in
            var raw: OpaquePointer?
            try check(git_status_list_new(&raw, handle, optsPtr))
            return StatusList(handle: raw!, repository: self)
        }
    }
}
```

### 6.3 `Repository.statusEntries(options:)`

```swift
public func statusEntries(
    options: StatusOptions = StatusOptions()
) throws(GitError) -> [StatusEntry] {
    let list = try statusList(options: options)
    // count / subscript each re-take the lock; cheap and matches Diff-style
    // snapshot materialization. `list` drops at return, freeing the handle.
    return (0..<list.count).map { list[$0] }
}
```

### 6.4 `Repository.status(forPath:)`

```swift
public func status(forPath path: String) throws(GitError) -> StatusFlags {
    try lock.withLock { () throws(GitError) -> StatusFlags in
        var flags: UInt32 = 0
        try path.withCString { cpath throws(GitError) in
            try check(git_status_file(&flags, handle, cpath))
        }
        return StatusFlags(rawValue: flags)
    }
}
```

### 6.5 `Repository.shouldIgnore(path:)`

```swift
public func shouldIgnore(path: String) throws(GitError) -> Bool {
    try lock.withLock { () throws(GitError) -> Bool in
        var ignored: Int32 = 0
        try path.withCString { cpath throws(GitError) in
            try check(git_status_should_ignore(&ignored, handle, cpath))
        }
        return ignored != 0
    }
}
```

### 6.7 Concurrency & lifetime

- `statusEntries` holds the repo lock only across `statusList(options:)`
  (which includes `git_status_list_new`). The `count` and each
  `subscript(Int)` call in the subsequent materialization loop each
  re-take the lock independently — same pattern as `Diff`.
- `StatusList` strong-references its `Repository` through the stored
  `repository: Repository` property. The repository therefore outlives
  the list; dropping the list frees the libgit2 handle before the
  repository is allowed to die.
- `StatusOptions.pathspec` and `.baseline` are only visible to libgit2
  during the `list_new` call. The `[String]` C-bridging and `Tree`
  `withExtendedLifetime` both terminate at the close of
  `withBridgedStatusOptions`.
