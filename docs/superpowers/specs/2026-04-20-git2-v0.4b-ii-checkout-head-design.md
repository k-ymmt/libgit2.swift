# Git2 Swift Wrapper — v0.4b-ii Checkout / HEAD Slice

**Status:** Draft
**Date:** 2026-04-20
**Scope:** Second slice of the v0.4b write-operations line. Adds the filesystem-touching surface: working-tree checkout (from HEAD, from an arbitrary tree / commit, and from the index), HEAD manipulation (attached and detached), and a high-level branch-switching convenience that composes the two. Builds on v0.2 (`docs/superpowers/specs/2026-04-20-git2-swift-wrapper-foundation-design.md`), v0.3 (`docs/superpowers/specs/2026-04-20-git2-v0.3-read-extensions-design.md`), v0.4a (`docs/superpowers/specs/2026-04-20-git2-v0.4a-write-foundation-design.md`), and v0.4b-i (`docs/superpowers/specs/2026-04-20-git2-v0.4b-i-index-design.md`).

## 1. Purpose

v0.4a covers ODB-write end-to-end. v0.4b-i covers the index / staging surface. What is still missing is the ability to move bytes onto the working tree and to move the HEAD pointer. v0.4b-ii closes that gap.

It serves two use cases jointly:

- **High-level branch switching.** A caller can say `try repo.checkout(branchNamed: "feature")` and have both the working tree and HEAD updated atomically, the way `git checkout <branch>` / `git switch <branch>` behave at the CLI.
- **Merge / rebase foundation.** The lower-level `git_checkout_head` / `git_checkout_tree` / `git_checkout_index` primitives are exposed directly, with HEAD manipulation kept orthogonal, so the eventual merge / cherry-pick / rebase slice (v0.5+) can compose them without a second refactor.

Together with v0.4a and v0.4b-i this completes the programmatic equivalent of the `git add` → `git commit` → `git checkout` / `git switch` loop, from public API, without reaching for `@_exported import Cgit2`.

v0.4b-ii is **additive**: every existing v0.2 / v0.3 / v0.4a / v0.4b-i public API keeps the same signature and behavior.

## 2. Target Platforms

Unchanged from v0.4b-i: Apple platforms only, macOS 13+, iOS 16+, Swift 6.2.

### 3.2 Explicitly not included in v0.4b-ii

Tracked in TODO.md under "Deferred from v0.4b-ii".

- **Checkout callbacks.** `notify_cb`, `progress_cb`, `perfdata_cb`. Bridging Swift closures to C callbacks (context pointer, lifetime) is its own implementation surface.
- **`CheckoutOptions` fields not surfaced.** `baseline`, `baseline_index`, `target_directory`, `ancestor_label` / `our_label` / `their_label`, `dir_mode`, `file_mode`, `file_open_flags`, `disable_filters`.
- **Merge-flavored strategy flags.** `GIT_CHECKOUT_USE_OURS`, `USE_THEIRS`, `SKIP_UNMERGED`. These land with the merge slice (v0.5+).
- **Not-yet-implemented libgit2 strategies.** `GIT_CHECKOUT_UPDATE_SUBMODULES*` — libgit2 itself does not implement them.
- **`checkoutTree(_: Tag, options:)` overload.** libgit2 accepts a tag as a treeish, but the v0.4b-ii overloads are `Tree` and `Commit` only.
- **`Repository.detachHead()`.** Sugar for `git_repository_detach_head`. Replaceable today with `setHead(detachedAt: try repo.head().target)`.
- **`setHead(detachedAtAnnotated:)`.** Wraps `git_repository_set_head_detached_from_annotated`. Annotated commits are not surfaced anywhere in v0.4b-ii.
- **Remote branches in `checkout(branchNamed:)`.** Local branches only (`GIT_BRANCH_LOCAL`).

### 3.3 Non-goals (unchanged from v0.4b-i)

- No shipped CLI.
- No 1:1 exposure of every libgit2 function; low-level access remains via `@_exported import Cgit2`.
- No Linux / Windows / Android support.
- No Keychain / credential-UI integration.
- No SHA-256 repositories.

## 4. Architectural Continuity

Every v0.2 / v0.3 / v0.4a / v0.4b-i decision carries forward unchanged.

- **Handle ownership.** No new handle type. Checkout and HEAD manipulation are actions on `Repository`; their outputs (trees, commits, references) are the v0.2–v0.4a types.
- **Parent–child lifetime.** Unchanged. `Index.checkout()` simply forwards to `self.repository`.
- **Typed throws.** Every new public API is `throws(GitError)`.
- **Per-`Repository` serialization.** Every new method takes `repository.lock.withLock` once at the top. Multi-step operations — notably `checkout(branch:)`'s "peel → `checkout_tree` → `set_head`" — run inside a single critical section so no other task can observe a mid-switch state.
- **Error representation.** `GitError` with `Code` / `Class`. No new error types.
- **Bootstrap / shutdown.** Unchanged.

### 4.1 Bare repository

libgit2's `git_checkout_*` reject bare repositories. v0.4b-ii does not add a Swift-layer pre-check; libgit2's error (typically `Class.repository`) propagates unchanged. Matches the v0.4a policy.

### 4.2 Unborn HEAD

- `checkoutHead()` on an unborn HEAD surfaces libgit2's `GIT_EUNBORNBRANCH` → `GitError.Code.unbornBranch`.
- `setHead(referenceName:)` against a not-yet-existing branch does **not** throw; libgit2 attaches HEAD to an unborn branch (this is how `git checkout --orphan` is implemented). Documented in the Swift API.

### 4.3 Shared index state

`Index.checkout(options:)` operates against the caller's `Index` handle, which — per v0.4b-i §4.1 — shares mutable state with any other `Index` derived from the same `Repository`. No new contract here; the existing documentation applies.

### 4.4 Branch-switch atomicity

`checkout(branch:)` runs `peel → checkout_tree → set_head` inside one `lock.withLock`. If `set_head` fails after `checkout_tree` succeeds, the working tree is switched but HEAD is stale. libgit2 does not roll back, and v0.4b-ii does not add Swift-layer rollback. This trade-off is documented; a rollback wrapper is deferred (see §10.2).

## 6. Types

### 6.1 `Repository.CheckoutOptions`

```swift
// Sources/Git2/Values/CheckoutOptions.swift

extension Repository {
    /// Options forwarded to libgit2's `git_checkout_*` family.
    ///
    /// The default value (`CheckoutOptions()`) maps to `GIT_CHECKOUT_SAFE`
    /// with no path filter — the same as what libgit2 uses when you pass
    /// `NULL` for the options.
    public struct CheckoutOptions: Sendable, Equatable {
        /// Checkout strategy flags. Empty set == `GIT_CHECKOUT_SAFE`.
        public var strategy: Strategy

        /// Optional pathspec. Empty array (default) means "all paths".
        ///
        /// Each entry is a wildmatch pattern against repository-relative
        /// paths (forward slashes, no leading separator). Pass
        /// `Strategy.disablePathspecMatch` in ``strategy`` to treat the
        /// list as exact matches.
        public var paths: [String]

        public init(strategy: Strategy = [], paths: [String] = [])
    }
}
```

### 6.2 `Repository.CheckoutOptions.Strategy`

```swift
extension Repository.CheckoutOptions {
    /// Checkout strategy flags. Bit values match `git_checkout_strategy_t`.
    public struct Strategy: OptionSet, Sendable, Equatable {
        public let rawValue: UInt32
        public init(rawValue: UInt32) { self.rawValue = rawValue }

        // GIT_CHECKOUT_SAFE == 0; the empty set `[]` expresses it.

        public static let force                  = Strategy(rawValue: 1 << 1)
        public static let recreateMissing        = Strategy(rawValue: 1 << 2)
        public static let allowConflicts         = Strategy(rawValue: 1 << 4)
        public static let removeUntracked        = Strategy(rawValue: 1 << 5)
        public static let removeIgnored          = Strategy(rawValue: 1 << 6)
        public static let updateOnly             = Strategy(rawValue: 1 << 7)
        public static let dontUpdateIndex        = Strategy(rawValue: 1 << 8)
        public static let noRefresh              = Strategy(rawValue: 1 << 9)
        public static let disablePathspecMatch   = Strategy(rawValue: 1 << 13)
        public static let skipLockedDirectories  = Strategy(rawValue: 1 << 18)
        public static let dontOverwriteIgnored   = Strategy(rawValue: 1 << 19)
        public static let conflictStyleMerge     = Strategy(rawValue: 1 << 20)
        public static let conflictStyleDiff3     = Strategy(rawValue: 1 << 21)
```

## 7. Public API

### 7.1 Low-level checkout

```swift
// Sources/Git2/Repository/Repository+Checkout.swift

extension Repository {
    /// Updates the working tree and index to match the commit HEAD points at.
    /// Equivalent to `git_checkout_head`.
    ///
    /// - Parameter options: Defaults to safe + no path filter.
    /// - Throws: ``GitError`` — ``GitError/Code/unbornBranch`` when HEAD is
    ///   unborn, a ``GitError/Class/repository`` error on a bare repo, and
    ///   the usual conflict errors when `options.strategy` is safe and the
    ///   working tree has divergent changes.
    public func checkoutHead(
        options: CheckoutOptions = CheckoutOptions()
    ) throws(GitError)

    /// Updates the working tree (and, by default, the index) to match `tree`.
    /// Equivalent to `git_checkout_tree`.
    ///
    /// Does **not** move HEAD. Pair with ``setHead(to:)`` (the `Commit`
    /// overload) or use ``checkout(branch:options:)`` when switching
    /// branches.
    public func checkoutTree(
        _ tree: Tree,
        options: CheckoutOptions = CheckoutOptions()
    ) throws(GitError)

    /// Updates the working tree to match the tree pointed at by `commit`.
    /// Equivalent to `git_checkout_tree` with a commit treeish.
    ///
    /// Does **not** move HEAD. See the `Tree` overload of
    /// ``checkoutTree(_:options:)``.
    public func checkoutTree(
        _ commit: Commit,
        options: CheckoutOptions = CheckoutOptions()
    ) throws(GitError)

    /// Updates the working tree to match `index`. Pass `nil` (the default)
    /// to use the repository's current index. Equivalent to
    /// `git_checkout_index` — where a `NULL` `index` argument means "use the
    /// repo's cached index".
    public func checkoutIndex(
        _ index: Index? = nil,
        options: CheckoutOptions = CheckoutOptions()
    ) throws(GitError)
}
```

```swift
// Sources/Git2/Index/Index+Checkout.swift

extension Index {
    /// Materializes this index into the parent repository's working tree.
    /// Sugar for `repository.checkoutIndex(self, options: ...)`.
    public func checkout(
        options: Repository.CheckoutOptions = Repository.CheckoutOptions()
    ) throws(GitError)
}
```

### 7.2 HEAD manipulation

```swift
// Sources/Git2/Repository/Repository+Head.swift

extension Repository {
    /// Points HEAD at a reference by canonical name.
    /// Equivalent to `git_repository_set_head`.
    ///
    /// libgit2 accepts a not-yet-existing branch name — HEAD becomes
    /// attached to an unborn branch. This is **not** an error and does not
    /// throw. libgit2 rejects names that resolve to a tree or blob.
    ///
    /// Does **not** touch the working tree. Pair with
    /// ``checkoutHead(options:)`` (or use ``checkout(branch:options:)``
    /// / ``checkout(branchNamed:options:)``) when switching branches.
    public func setHead(referenceName: String) throws(GitError)

    /// Points HEAD directly at the commit with the given OID (detached).
    /// Equivalent to `git_repository_set_head_detached`.
    ///
    /// - Throws: ``GitError`` — ``GitError/Code/notFound`` if no commit
    ///   exists at `oid`.
    public func setHead(detachedAt oid: OID) throws(GitError)

    /// Sugar for ``setHead(referenceName:)`` using `reference.name`.
    public func setHead(to reference: Reference) throws(GitError)

    /// Sugar for ``setHead(detachedAt:)`` using `commit.oid` (detached).
    public func setHead(to commit: Commit) throws(GitError)
}
```

### 7.3 High-level branch switching

```swift
// Sources/Git2/Repository/Repository+Checkout.swift  (same file as §7.1)

extension Repository {
    /// Switches to `branch`: updates the working tree to its tree, then
    /// moves HEAD to point at it.
    ///
    /// Both steps run inside a single critical section. Another task cannot
    /// observe a state where the working tree matches the new branch but
    /// HEAD still points at the old one.
    ///
    /// If the first step succeeds but libgit2's `git_repository_set_head`
    /// then fails, the working tree is switched while HEAD is not. libgit2
    /// does not roll back, and neither does this wrapper. Check errors from
    /// this call and resolve manually if you hit that window.
    ///
    /// Rejects references that are not local branches up-front (before
    /// touching the working tree). Passing a tag, remote branch, or other
    /// non-branch reference throws ``GitError/Code/invalidSpec`` with
    /// ``GitError/Class/reference``. For checkout-without-moving-HEAD of a
    /// tag / arbitrary commit, use ``checkoutTree(_:options:)`` with the
    /// `Commit` overload instead.
    ///
    /// - Parameters:
    ///   - branch: A local branch reference (typically from
    ///     ``references`` or ``createBranch(named:at:force:)``).
    ///   - options: Defaults to safe + no path filter.
    /// - Throws: ``GitError``.
    public func checkout(
        branch: Reference,
        options: CheckoutOptions = CheckoutOptions()
    ) throws(GitError)

    /// Switches to the local branch with the given short name. Sugar for
    /// `git_branch_lookup(GIT_BRANCH_LOCAL)` + ``checkout(branch:options:)``.
    ///
    /// - Parameter name: Short branch name (e.g. `"feature"`), no
    ///   `refs/heads/` prefix.
    /// - Throws: ``GitError`` — ``GitError/Code/notFound`` when the branch
    ///   does not exist, or any error surfaced by
    ///   ``checkout(branch:options:)``.
    public func checkout(
        branchNamed name: String,
        options: CheckoutOptions = CheckoutOptions()
    ) throws(GitError)
}
```

## 8. Internal Infrastructure

### 8.1 `git_strarray` bridge

```swift
// Sources/Git2/Internal/StrArrayBridge.swift

/// Invokes `body` with a `git_strarray *` view of `paths` — or `nil` if
/// `paths` is empty (libgit2 treats that as "all paths").
///
/// The strarray and its backing C strings are valid only for the duration
/// of the closure. Callers must not retain the pointer.
internal func withGitStrArray<R>(
    _ paths: [String],
    _ body: (UnsafePointer<git_strarray>?) throws(GitError) -> R
) throws(GitError) -> R
```

- `paths.isEmpty` → invokes `body(nil)`.
- Otherwise: allocates a `ContiguousArray<UnsafeMutablePointer<CChar>?>`, `strdup`s each element, wraps it into a stack-allocated `git_strarray`, invokes `body(&arr)`, then frees every element and the backing array in a `defer` so cleanup runs even if `body` throws.
- Typed as `throws(GitError)` so the outer `withCOptions` helper (§8.2) can forward throws through without wrapping / unwrapping.

### 8.2 `CheckoutOptions` → `git_checkout_options` bridge

```swift
// Sources/Git2/Internal/CheckoutOptionsBridge.swift

extension Repository.CheckoutOptions {
    /// Builds a `git_checkout_options`, initializes it via
    /// `git_checkout_options_init`, applies `strategy` and `paths`, and
    /// invokes `body`. The `paths` strarray's lifetime is scoped to `body`.
    internal func withCOptions<R>(
        _ body: (UnsafePointer<git_checkout_options>) throws(GitError) -> R
    ) throws(GitError) -> R
}
```

- `git_checkout_options_init(&opts, UInt32(GIT_CHECKOUT_OPTIONS_VERSION))`.
- `opts.checkout_strategy = strategy.rawValue`.
- `paths` is bridged via §8.1. The resulting `UnsafePointer<git_strarray>?` has its `pointee` copied into `opts.paths` (libgit2 reads the strarray only during the call; the buffer lives as long as `body` runs).
- Callback fields / baseline fields / label fields / mode fields stay at their default-init values — i.e. libgit2's defaults.

### 8.3 Lock discipline

Every new public method takes `lock.withLock` **exactly once** at the top, then performs all libgit2 calls inside that scope. For `checkout(branch:)` and `checkout(branchNamed:)`, all of:

1. `git_branch_lookup(...)` (only for `branchNamed:`)
2. `git_reference_is_branch(ref) != 0` — else throw `GitError(code: .invalidSpec, class: .reference)` **before** touching the working tree. (`checkout(branchNamed:)` never hits this throw because `git_branch_lookup(GIT_BRANCH_LOCAL)` already guarantees a local-branch ref; kept in the shared helper for the `checkout(branch:)` caller.)
3. `git_reference_peel(&obj, ref, GIT_OBJECT_COMMIT)`
4. `git_checkout_tree(repo, obj, &opts)`
5. `git_repository_set_head(repo, git_reference_name(ref))`
6. `git_object_free(obj)` and `git_reference_free(ref)` (for the `branchNamed:` path)

happen under the single lock acquisition. A shared internal helper `performCheckout(branchHandle:options:)` assumes "lock is held" and is called from both public entry points to avoid duplicating steps 2–6.

### 8.4 `setHead` overload internals

To avoid reentrant `withLock`, the `Reference` and `Commit` overloads do **not** call the string / OID overloads; they take the lock once and call libgit2 directly:

- `setHead(to reference: Reference)` → `git_repository_set_head(handle, git_reference_name(reference.handle)!)`.
- `setHead(to commit: Commit)` → copies `git_commit_id(commit.handle)!.pointee` into a `git_oid` and calls `git_repository_set_head_detached(handle, &oid)`.

### 9.2 Error-code specificity

Same policy as v0.3 / v0.4a / v0.4b-i: every failure path that maps to a specific `Code` / `Class` is asserted by `#expect(error.code == .X)` / `#expect(error.class == .Y)`, not just by `throws: GitError.self`.

### 9.3 Fixture additions

None. Existing `TestFixture.makeLinearHistory` / `makeMergeHistory` / `makeBranches` / `makeCommitWithTree` cover the scenarios (branch switching = `makeLinearHistory` + `makeBranches`; checkout-from-tree = `makeLinearHistory` with multiple commits).

### 9.4 Known test-coverage gaps (accepted)

- **Symlink / executable-bit checkout.** Apple-platform behavior is the only thing in scope, and `.link` / submodule (`.commit`) round-trip coverage is tracked under the v0.4a "FileMode integration coverage" item.
- **Full wildmatch semantics for `paths`.** A couple of glob cases (`*.swift`) and an exact-match case with `disablePathspecMatch` are exercised; libgit2's full wildmatch spec is not re-tested.
- **Callback behavior.** `notify_cb` / `progress_cb` are not surfaced, so no tests.
- **Cross-repository misuse.** `checkoutIndex(otherRepoIndex)` is not type-prevented. libgit2 surfaces a runtime error; the Swift layer does not add an assertion.

## 10. Roadmap

### 10.1 Immediate follow-up

v0.4b-ii completes the v0.4 line (a + b-i + b-ii). The next slice is v0.5 (network / merge):

- **Remote / fetch / push** (HTTPS).
- **Merge / rebase / cherry-pick.** Exercises the merge-flavored checkout flags (`USE_OURS` / `USE_THEIRS` / `SKIP_UNMERGED`) that v0.4b-ii deliberately does not surface.

### 10.2 Deferred from v0.4b-ii (additive, non-breaking)

Tracked in TODO.md.

- **Checkout callbacks.** `notify_cb` / `progress_cb` / `perfdata_cb`. Land when a concrete UI need appears.
- **Checkout-options fields.** `baseline` / `baseline_index` / `target_directory` / `ancestor_label` / `our_label` / `their_label` / `dir_mode` / `file_mode` / `file_open_flags` / `disable_filters`. `baseline` in particular deserves a second look during the merge slice.
- **Merge-flavored strategy flags.** `useOurs` / `useTheirs` / `skipUnmerged` — part of the merge slice.
- **`checkoutTree(_: Tag, options:)` overload.** libgit2 accepts tags as treeishes. Not covered in v0.4b-ii.
- **`Repository.detachHead()`.** Sugar for `git_repository_detach_head`. Replaceable today with `setHead(detachedAt: try repo.head().target)`.
- **`setHead(detachedAtAnnotated:)`.** Wraps `git_repository_set_head_detached_from_annotated`. Useful for richer reflog messages once annotated commits are surfaced.
- **Remote-branch switching.** `GIT_BRANCH_REMOTE` side of `checkout(branchNamed:)`.
- **Atomic branch-switch rollback.** If `checkout_tree` succeeds but `set_head` fails, the wrapper currently leaves the working tree switched and HEAD stale. A rollback wrapper ("remember old HEAD, reset on failure") is deferred — complexity high, concrete demand not yet surfaced.
- **Checkout FileMode integration coverage.** `.link` / `.commit` (submodule) round-trip through checkout, aligned with the v0.4a TODO item of the same shape.

### 10.3 Versioning

v0.4b-ii ships as a minor bump (e.g. v0.4.2) — purely additive on top of v0.4b-i (v0.4.1). That closes out the v0.4 write-operations line. The "b-i / b-ii" suffixes in this design are for tracking only — not intended as release tags.
