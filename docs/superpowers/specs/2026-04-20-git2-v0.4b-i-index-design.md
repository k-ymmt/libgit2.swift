# Git2 Swift Wrapper — v0.4b-i Index Slice

**Status:** Draft
**Date:** 2026-04-20
**Scope:** First slice of the v0.4b write-operations line. Adds the minimum surface required to read and mutate the repository index (`.git/index`), stage files from the working directory, inspect merge conflicts, and write a tree object from the current index state. Builds on v0.2 (`docs/superpowers/specs/2026-04-20-git2-swift-wrapper-foundation-design.md`), v0.3 (`docs/superpowers/specs/2026-04-20-git2-v0.3-read-extensions-design.md`), and v0.4a (`docs/superpowers/specs/2026-04-20-git2-v0.4a-write-foundation-design.md`).

## 1. Purpose

v0.2 / v0.3 cover the full read-only surface; v0.4a covers the ODB-write surface end-to-end (blobs → trees → commits → refs). What is still missing is the bridge between the working directory and the ODB — the Git index.

v0.4b-i adds the smallest set of APIs that can express:

- opening a repository's index,
- enumerating index entries and their stages,
- inspecting merge conflicts (read-only),
- staging and unstaging paths from the working directory,
- persisting in-memory index changes to `.git/index`,
- reloading the in-memory index from disk, and
- writing a tree object to the ODB from the current index state.

Together with v0.4a this is enough to express the full `git add` → `git commit` flow from public API, without dropping to `@_exported import Cgit2`.

Working-directory mutation (checkout) and HEAD manipulation remain out of scope — they are the v0.4b-ii slice.

v0.4b-i is **additive**: every existing v0.2 / v0.3 / v0.4a public API keeps the same signature and behavior.

## 2. Target Platforms

Unchanged from v0.4a: Apple platforms only, macOS 13+, iOS 16+, Swift 6.2.

## 3. Scope

### 3.1 Included in v0.4b-i

| Area | Additions |
|---|---|
| Values | `IndexEntry`, `IndexEntry.Stage`, `IndexConflict` |
| Index (new handle class) | `entries`, `hasConflicts`, `conflicts`, `entry(at:stage:)`, `conflict(for:)`, `addPath(_:)`, `removePath(_:)`, `save()`, `reload(force:)`, `writeTree()` |
| Repository | `index()` |

### 3.2 Explicitly not included in v0.4b-i

- **Checkout / working-directory manipulation.** `git_checkout_head`, `git_checkout_tree`, `git_checkout_index`. v0.4b-ii.
- **HEAD manipulation.** `git_repository_set_head`, `git_repository_set_head_detached`, branch switching. v0.4b-ii.
- **Manual entry construction.** A public API that hands libgit2 a caller-built `IndexEntry` (stage included) and inserts it into the index. Deferred until Merge / cherry-pick need it.
- **`index.readTree(from:)` and `index.clear()`.** Same rationale — consumed by Merge / reset-style flows that do not exist yet.
- **Pathspec-based bulk operations.** `git_index_add_all`, `git_index_remove_all`, `git_index_update_all`. v0.4b-i takes a single `String` path per call.
- **Extended `IndexEntry` stat fields.** `mtime`, `ctime`, `fileSize`, `uid`, `gid`, `dev`, `ino`, `flags_extended`. Not needed for stage / write-tree / conflict-read use cases.
- **Lazy `Sequence` iteration over entries / conflicts.** v0.4b-i returns snapshot `Array`s.
- **Auto-refresh semantics.** `Index` does not detect on-disk `.git/index` changes automatically — the caller chooses when to `reload`.

### 3.3 Non-goals (unchanged from v0.4a)

- No shipped CLI.
- No 1:1 exposure of every libgit2 function; low-level access remains via `@_exported import Cgit2`.
- No Linux / Windows / Android support.
- No Keychain / credential-UI integration.
- No SHA-256 repositories.

## 4. Architectural Continuity

Every v0.2 / v0.3 / v0.4a decision carries forward unchanged:

- **Hybrid resource ownership.** The new `Index` class is a libgit2-handle wrapper in the same shape as `Reference` / `Commit` — it owns a `git_index *`, frees it in `deinit`, and holds a strong reference to its owning `Repository`.
- **Parent-child lifetime.** `Index` stores `let repository: Repository` so the repository outlives anything derived from it. `IndexEntry` and `IndexConflict` are snapshot values and carry no repo tie.
- **Typed throws.** Every write API uses `throws(GitError)`. Read-only accessors that cannot fail (e.g., `hasConflicts`) remain non-throwing.
- **Per-`Repository` serialization.** All libgit2 calls — reads, mutations, and I/O — run inside `repository.lock.withLock`. Multi-step flows (`git_index_write_tree` → `git_tree_lookup`, conflict-iterator enumeration) are held inside a single critical section.
- **Error representation.** Still `GitError` with `Code` / `Class` enums; no new error types.
- **Lifecycle.** Still `Git.bootstrap()` / `Git.shutdown()`.

### 4.1 Shared index state

libgit2's `git_repository_index` returns a refcounted object that is cached per-repository: two calls against the same `Repository` return the same underlying `git_index *`. v0.4b-i surfaces this honestly — the spec documents that two `Index` instances derived from one `Repository` share mutable state, and a mutation through one is visible through the other. `Index.reload(force:)` is the caller's tool when another process may have rewritten `.git/index` between operations.

## 6. Types

### 6.1 `IndexEntry.Stage`

Models the Git merge-stage flag encoded in the lower two bits of `git_index_entry.flags`.

```swift
// Sources/Git2/Values/IndexEntry.swift

extension IndexEntry {
    /// Merge stage of an index entry.
    public enum Stage: Sendable, Equatable {
        /// stage 0 — resolved / non-conflicted.
        case normal
        /// stage 1 — common ancestor side during a merge.
        case ancestor
        /// stage 2 — "our" side during a merge.
        case ours
        /// stage 3 — "their" side during a merge.
        case theirs
    }
}
```

libgit2 guarantees stage values are in `0...3`. The internal `Stage.init(flags:)` applies `GIT_INDEX_ENTRY_STAGEMASK` + `GIT_INDEX_ENTRY_STAGESHIFT` and hits `preconditionFailure` on anything outside that range (unreachable — a libgit2 contract violation).

A public `rawFlags` round-trip is intentionally not provided in v0.4b-i because no public API accepts a user-constructed `IndexEntry` back into the index (see §3.2). It ships with the deferred manual-entry API.

### 6.2 `IndexEntry`

```swift
// Sources/Git2/Values/IndexEntry.swift

/// A single entry in the repository index.
public struct IndexEntry: Sendable, Equatable {
    /// The entry path, relative to the repository working directory.
    public let path: String

    /// The OID of the staged blob (or sub-tree / submodule commit).
    public let oid: OID

    /// The entry filemode.
    public let filemode: TreeEntry.FileMode

    /// The merge stage.
    public let stage: Stage

    public init(
        path: String,
        oid: OID,
        filemode: TreeEntry.FileMode,
        stage: Stage
    )
}
```

Reusing `TreeEntry.FileMode` (public since v0.3) keeps the filemode vocabulary identical between tree and index surfaces. The public initializer exists primarily to let tests build and compare `IndexEntry` values without touching the libgit2 surface.

### 6.3 `IndexConflict`

```swift
// Sources/Git2/Values/IndexConflict.swift

/// The three-way side collection for one conflicted path in the index.
public struct IndexConflict: Sendable, Equatable {
    /// The conflicting path (shared by all non-nil sides).
    public let path: String

    /// The common-ancestor entry (stage 1), or `nil` (e.g. add/add conflicts).
    public let ancestor: IndexEntry?

    /// The "ours" entry (stage 2), or `nil` (e.g. modify/delete conflicts
    /// where we deleted).
    public let ours: IndexEntry?

    /// The "theirs" entry (stage 3), or `nil` (e.g. modify/delete conflicts
    /// where they deleted).
    public let theirs: IndexEntry?

    public init(
        path: String,
        ancestor: IndexEntry?,
        ours: IndexEntry?,
        theirs: IndexEntry?
    )
}
```

At least one of `ancestor` / `ours` / `theirs` is non-nil in any conflict that libgit2 returns — `IndexConflict` with all three nil cannot originate from the index iterator. The struct does not enforce this at the type level (public initializer accepts any combination) so tests can build degenerate values freely; runtime-origin `IndexConflict`s always respect the invariant.

## 7. Public API

### 7.1 `Repository.index()`

```swift
// Sources/Git2/Repository/Repository+Index.swift

extension Repository {
    /// Opens the repository's index (`.git/index`).
    ///
    /// libgit2 refcounts a single index object per repository. Two ``Index``
    /// values obtained from the same ``Repository`` share the underlying
    /// mutable state — a mutation through one is visible through the other.
    /// Call ``Index/reload(force:)`` if another process may have rewritten the
    /// on-disk `.git/index` file after your last call.
    ///
    /// - Throws: ``GitError`` — rare in practice; surfaces any libgit2
    ///   failure acquiring the cached index pointer.
    public func index() throws(GitError) -> Index
}
```

Implementation: `git_repository_index(&raw, handle)` under `lock.withLock`, wrap `raw!` in `Index(handle:repository:)`.

### 7.2 `Index` handle class

```swift
// Sources/Git2/Index/Index.swift

/// The repository index (the staging area between the working tree and the ODB).
///
/// ``Index`` owns a libgit2 `git_index *` handle. It is a mutable, long-lived
/// handle — unlike ``Tree`` and ``Commit``, which are immutable snapshots. All
/// access is serialized through the parent ``Repository``'s internal lock.
///
/// Two ``Index`` values obtained from the same ``Repository`` share their
/// underlying mutable state (see ``Repository/index()``).
public final class Index: @unchecked Sendable {
    internal let handle: OpaquePointer

    /// The repository this index belongs to.
    public let repository: Repository

    internal init(handle: OpaquePointer, repository: Repository)

    deinit { git_index_free(handle) }
}
```

### 7.3 Read-only accessors

```swift
extension Index {
    /// A point-in-time snapshot of every entry currently in the index.
    ///
    /// Entries are returned in libgit2's internal order (path-sorted with
    /// stage tiebreaking). Mutating the index through another operation
    /// invalidates this snapshot — re-read the property to get fresh data.
    public var entries: [IndexEntry] { get }

    /// Whether the index currently records any merge conflicts
    /// (i.e. entries with stage other than `.normal`).
    public var hasConflicts: Bool { get }

    /// A point-in-time snapshot of every conflicting path.
    public var conflicts: [IndexConflict] { get }

    /// Looks up a single entry by path and stage.
    ///
    /// - Parameter stage: Defaults to `.normal`. For conflicted paths, pass
    ///   `.ancestor` / `.ours` / `.theirs` to reach the specific side — or
    ///   use ``conflict(for:)`` / ``conflicts`` to retrieve all three at once.
    /// - Returns: The matching entry, or `nil` if no entry exists at that path
    ///   and stage.
    public func entry(
        at path: String,
        stage: IndexEntry.Stage = .normal
    ) -> IndexEntry?

    /// Looks up the three-way conflict for a single path.
    ///
    /// - Returns: An ``IndexConflict`` with the non-nil sides libgit2 has on
    ///   record, or `nil` if `path` has no conflict.
    public func conflict(for path: String) -> IndexConflict?
}
```

Implementation notes:

- `entries` loops `git_index_entrycount` → `git_index_get_byindex` inside a single `lock.withLock`, mapping each `git_index_entry *` into an `IndexEntry` before releasing the lock.
- `conflicts` wraps `git_index_conflict_iterator_new` / `_next` / `_iterator_free`. All three calls run under the same lock acquisition.
- `entry(at:stage:)` is `git_index_get_bypath`. The function is documented by libgit2 as non-failing (returns `NULL` on miss); Swift returns `nil`.
- `conflict(for:)` calls `git_index_conflict_get`. If libgit2 returns `0` the method builds an `IndexConflict` from the three returned pointers (any of which may be `NULL`) and returns it. If libgit2 returns `GIT_ENOTFOUND` (the path is not in conflict), Swift returns `nil` — this is the only miss code that is suppressed. Any other non-zero return value is a libgit2 contract violation and raises `preconditionFailure`; the method stays non-throwing so callers do not have to branch on an error path that cannot occur against a valid index handle.

### 7.4 Mutation

```swift
extension Index {
    /// Stages the file at `path` (relative to the repository working directory)
    /// into the index.
    ///
    /// - Parameter path: Path relative to the repository working directory.
    ///   Forward slashes only, no leading separator.
    /// - Throws: ``GitError`` — typically ``GitError/Code/notFound`` if the
    ///   file doesn't exist, or a ``GitError/Class/repository`` error on a
    ///   bare repository.
    public func addPath(_ path: String) throws(GitError)

    /// Removes the entry at `path` from the index.
    ///
    /// Does not touch the working directory. For conflicted paths, removes
    /// every stage (ancestor / ours / theirs) in one call.
    ///
    /// - Throws: ``GitError``.
    public func removePath(_ path: String) throws(GitError)
}
```

- `addPath` is `git_index_add_bypath`. The file at `path` must exist in the working directory; libgit2 will stat it, hash its contents into a new blob, and record the entry. On a bare repository libgit2 surfaces an error and Swift propagates it unchanged (no Swift-layer pre-check — same policy as v0.4a).
- `removePath` is `git_index_remove_bypath`. libgit2's `_bypath` variant also strips every stage for conflicted paths, which is what a UI that lets a user "drop" a conflicted path expects.

### 7.5 I/O

```swift
extension Index {
    /// Reloads the index from the on-disk `.git/index` file, discarding any
    /// unsaved in-memory changes.
    ///
    /// - Parameter force: If `true`, reload unconditionally. If `false`
    ///   (default), libgit2 may short-circuit when the on-disk file's
    ///   modification-time stamp matches the cached one.
    /// - Throws: ``GitError``.
    public func reload(force: Bool = false) throws(GitError)

    /// Persists the in-memory index to the on-disk `.git/index` file.
    ///
    /// - Throws: ``GitError`` — typically a ``GitError/Class/index`` error if
    ///   the index cannot be serialized.
    public func save() throws(GitError)

    /// Writes a tree object to the ODB from the current index state.
    ///
    /// Does **not** modify the on-disk `.git/index`. Call ``save()`` separately
    /// if you want the current in-memory index persisted.
    ///
    /// - Throws: ``GitError`` — surfaces libgit2's `.unmerged` / index-dirty
    ///   errors if the index has conflicts or other unwritable state.
    public func writeTree() throws(GitError) -> Tree
}
```

Implementation:

- `reload(force:)` — `git_index_read(handle, force ? 1 : 0)`.
- `save()` — `git_index_write(handle)`.
- `writeTree()` — `git_index_write_tree(&oid, handle)` → `git_tree_lookup(&tree, repo.handle, &oid)` → wrap in `Tree(handle:repository:)`. Both libgit2 calls run under a single `lock.withLock`, so the OID is guaranteed to be in the ODB before the lookup.

## 8. Internal Infrastructure

### 8.1 `IndexEntry.init(_:)` (internal)

```swift
extension IndexEntry {
    internal init(_ raw: UnsafePointer<git_index_entry>) {
        let r = raw.pointee
        self.path = String(cString: r.path)
        self.oid = OID(raw: r.id)
        self.filemode = TreeEntry.FileMode.from(git_filemode_t(r.mode))
        self.stage = Stage(flags: r.flags)
    }
}
```

`TreeEntry.FileMode.from(_:)` already exists as an internal `git_filemode_t` → enum mapper (added in v0.3, re-used by v0.4a's `tree(entries:)` via the `.raw` round-trip helper). `git_index_entry.mode` is typed `uint32_t` in libgit2's public header and is cast to `git_filemode_t` at the call site.

### 8.2 `IndexEntry.Stage.init(flags:)` (internal)

```swift
extension IndexEntry.Stage {
    internal init(flags: UInt16) {
        let masked = (Int32(flags) & GIT_INDEX_ENTRY_STAGEMASK) >> GIT_INDEX_ENTRY_STAGESHIFT
        switch masked {
        case 0: self = .normal
        case 1: self = .ancestor
        case 2: self = .ours
        case 3: self = .theirs
        default: preconditionFailure("libgit2 returned out-of-range stage value: \(masked)")
        }
    }
}
```

### 8.3 Lock discipline

Every `Index` method takes `repository.lock.withLock` once at the top and performs all libgit2 calls — including multi-step flows like the conflict-iterator loop and `write_tree` + `tree_lookup` — inside that single call. This mirrors the discipline v0.2 / v0.3 / v0.4a established and keeps partial state invisible to concurrent readers.

No cross-file internal helper (of the `signatureHandle(for:)` shape from v0.4a) is introduced — `Index` mutations do not share helper C-level allocations with any other extension.

### 9.2 Error-code specificity

Every failure path that maps to a specific `GitError.Code` or `Class` is asserted by code, not just by throw-type. Consistent with v0.3 / v0.4a tightening.

### 9.3 `TestFixture` additions

v0.4b-i does not rewrite any existing `TestFixture` helper. It adds one:

```swift
extension TestFixture {
    /// Populates the repository's index with a synthetic three-way conflict
    /// on `path`. Uses libgit2 directly (`git_index_add` with stage-encoded
    /// flags) because v0.4b-i does not expose a public API for writing
    /// stage 1 / 2 / 3 entries. Will migrate to the public Merge API once
    /// that slice lands.
    func makeConflictedIndex(
        at path: String,
        ancestor: Data?,
        ours: Data?,
        theirs: Data?
    ) throws
}
```

The helper builds `git_index_entry` values, sets `flags` with the appropriate `GIT_INDEX_ENTRY_STAGESHIFT`-encoded stage, and calls `git_index_add`. Only non-nil sides are inserted, so modify/delete-style conflicts are expressible by passing `nil` for one side.

### 9.4 Regression coverage via existing v0.4a flow

Existing `TestFixture.makeLinearHistory` / `makeAnnotatedTag` / `makeBranches` continue to go through the v0.4a public API unchanged; `IndexMutationTests` exercises `addPath` → `save` → `writeTree` → `repo.commit(tree:)`, transitively verifying that the v0.4a commit path composes correctly with the v0.4b-i index path.

### 9.5 Known test-coverage gaps (accepted)

- **Working-tree `stat` integrity.** The `addPath` code path reads mtime / ctime / size from the OS. Fine-grained assertions on those fields are not part of v0.4b-i (they are not public fields; see §6.2 scope). Coverage stays functional.
- **Very large index.** No performance measurement. Functional correctness only via smoke test.
- **Non-UTF-8 paths.** libgit2 handles paths as byte strings. Swift `String` round-trip assumes UTF-8. Apple platforms use UTF-8 NFC by convention, so this is not exercised.
- **Cross-repository misuse.** Passing an `Index` whose parent `Repository` differs from the target of a downstream call is not type-prevented. libgit2 surfaces a runtime error; v0.4b-i does not add a Swift-layer assertion.

## 10. Roadmap

### 10.1 Immediate follow-up (v0.4b-ii)

- **Checkout.** `git_checkout_head`, `git_checkout_tree`, `git_checkout_index` with safety options. Working-directory mutation.
- **HEAD manipulation.** `git_repository_set_head`, `git_repository_set_head_detached`.
- **Branch switching convenience.** `set_head` + `checkout_head` combined API — shape decided at v0.4b-ii design time.

### 10.2 Deferred from v0.4b-i (additive, non-breaking)

- **Manual `IndexEntry` insertion.** Public API that accepts a caller-constructed `IndexEntry` (including a public `Stage.rawFlags` round-trip) and writes it to the index. Lands with the Merge slice.
- **`Index.readTree(from:)` and `Index.clear()`.** Consumed by Merge / reset-style flows; premature without them.
- **Pathspec-based bulk operations.** `addAll(pathspec:)`, `removeAll(pathspec:)`, `updateAll(pathspec:)` wrapping `git_index_add_all` / `_remove_all` / `_update_all`.
- **Extended `IndexEntry` fields.** `mtime`, `ctime`, `fileSize`, `uid`, `gid`, `dev`, `ino`, `flags_extended`. Expose when a stat-based diff or working-tree-dirty-detection API surfaces.
- **Lazy iteration.** `lazyEntries() -> some Sequence<IndexEntry>`, `lazyConflicts() -> some Sequence<IndexConflict>`. Add if a concrete performance pressure appears.
- **`TestFixture.makeConflictedIndex` migration to public Merge API.** Closes the last `import Cgit2` path in the fixture layer (parallel to v0.4a's rewrite of `makeLinearHistory` onto v0.4a public API).

### 10.3 Versioning

v0.4b-i ships as a minor bump (e.g. v0.4.1) — purely additive. v0.4b-ii, once landed, is the next minor bump. The "i / ii" suffixes in this design document are for tracking only — they are not intended as release tags.
