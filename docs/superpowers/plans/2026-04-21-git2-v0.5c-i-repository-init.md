# v0.5c-i Repository.create Porcelain Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the first "git CLI porcelain catch-up" slice:
`Repository.create(at:bare:initialBranch:)` as a thin wrapper over
`git_repository_init_ext`, plus migrate every existing test-layer
`git_repository_init` call site (~18 sites) off `@testable import
Cgit2` onto the new public API.

**Architecture:** One new public static factory on `Repository`,
mirroring the shape and bootstrap-gate of `Repository.open`. Sets
`GIT_REPOSITORY_INIT_MKPATH` unconditionally so missing path
components are created. Delegates initial-branch resolution to
libgit2 (reads `init.defaultBranch`, falls back to `"master"`) to
preserve the project's thin-wrapper policy. Test-layer migration
is mechanical: each existing `git_repository_init` + `Repository.open`
pair collapses to a single `Repository.create` call.

**Tech Stack:** Swift 6.2, libgit2 1.9.x (via XCFramework), Apple
platforms only. Test framework: `swift-testing`.

**Source of truth:**
`docs/superpowers/specs/2026-04-21-git2-v0.5c-i-repository-init-design.md`
(commit `2e7d936`).

```swift
import Cgit2
import Foundation

extension Repository {
    /// Creates (or reinitializes) a Git repository at `url`.
    ///
    /// Thin wrapper over `git_repository_init_ext` with
    /// `GIT_REPOSITORY_INIT_MKPATH` pre-set, so missing intermediate
    /// directories under `url` are created automatically.
    ///
    /// Reinitializing an existing repository is safe and idempotent:
    /// libgit2 re-applies config but preserves the existing object
    /// database, refs, and HEAD. HEAD is re-targeted only when
    /// `initialBranch` is explicitly non-`nil` on the reinit call.
    ///
    /// - Parameters:
    ///   - url: Working-tree directory for non-bare repositories; the
    ///     repository root directory for bare repositories.
    ///   - bare: `true` to create a bare repository (no working tree).
    ///   - initialBranch: Name of the initial branch written into HEAD
    ///     (without the `refs/heads/` prefix â€” libgit2 adds it). `nil`
    ///     delegates to libgit2, which reads `init.defaultBranch` from
    ///     the user's gitconfig and falls back to `"master"`.
    /// - Returns: An opened ``Repository`` handle for the new
    ///   (or reinitialized) repository.
    /// - Throws: ``GitError`` â€” ``GitError/Code/invalidSpec`` when `url`
    ///   yields no file-system representation; libgit2 code propagation
    ///   otherwise.
    /// - Precondition: ``Git/bootstrap()`` has been called.
    public static func create(
        at url: URL,
        bare: Bool = false,
        initialBranch: String? = nil
    ) throws(GitError) -> Repository {
        GitRuntime.shared.requireBootstrapped()

        var opts = git_repository_init_options()
        let initRC = git_repository_init_options_init(
            &opts,
            UInt32(GIT_REPOSITORY_INIT_OPTIONS_VERSION)
        )
        precondition(
            initRC == 0,
            "git_repository_init_options_init failed (libgit2 version mismatch?)"
        )

        var flags = UInt32(GIT_REPOSITORY_INIT_MKPATH.rawValue)
        if bare {
            flags |= UInt32(GIT_REPOSITORY_INIT_BARE.rawValue)
        }
        opts.flags = flags

        var raw: OpaquePointer?
        let result: Int32 = url.withUnsafeFileSystemRepresentation { path -> Int32 in
            guard let path else { return GIT_EINVALIDSPEC.rawValue }
            if let branch = initialBranch {
                return branch.withCString { branchPtr -> Int32 in
                    opts.initial_head = branchPtr
                    return git_repository_init_ext(&raw, path, &opts)
                }
            } else {
                opts.initial_head = nil
                return git_repository_init_ext(&raw, path, &opts)
            }
        }
        try check(result)
        return Repository(handle: raw!)
    }
}
```

## Task 5: Error-path test #8 (invalid `initialBranch`)

**Scope note (revised after empirical findings in implementation):**
The original plan included a second test (`create_invalidUrl_throwsInvalidSpec`)
exercising the `GIT_EINVALIDSPEC` Swift-layer sentinel. Empirical testing
on macOS showed that `URL.withUnsafeFileSystemRepresentation` never
returns `nil` for any URL shape we tried (`http://`, `ftp://`,
`mailto:`, malformed strings). The sentinel is therefore dead code
on the supported platforms and cannot be driven from a test.
`Repository.open(at:)` carries the identical sentinel and `RepositoryOpenTests`
likewise does not exercise it; `Repository.create` now matches that
parity. Spec Â§7.3 documents this. Task 5 accordingly ships a single
test â€” what was #9 in the original plan, now renumbered #8.

**Empirical note on `initialBranch: "HEAD"`:** libgit2 1.9.x accepts
`"HEAD"` as a branch name (writes `HEAD = "ref: refs/heads/HEAD"`).
Use a name that universally violates `git check-ref-format` â€” the
steps below start with `"foo bar"` (embedded space) as the primary
candidate.

## Notes for the implementer

- **Bootstrap pattern.** Every test in `RepositoryCreateTests` calls
  `try Git.bootstrap(); defer { try? Git.shutdown() }` and is nested
  inside `extension RuntimeSensitiveTests { @Suite(.serialized) ... }`.
  This mirrors every other test in the codebase that exercises
  `Repository` APIs. Copy the pattern from `RepositoryOpenTests.swift`
  verbatim.
- **`withTemporaryDirectory` vs `withTemporaryDirectoryAsync`.** All
  new tests in this plan use the sync variant. If you port any test
  that uses the async variant (none in this plan do), keep the
  async helper.
- **`gitDirectory.appendingPathComponent("HEAD")`.** For bare repos,
  `gitDirectory` is the repo root and HEAD is at `<root>/HEAD`. For
  non-bare, `gitDirectory` is `<workdir>/.git/` and HEAD is at
  `<workdir>/.git/HEAD`. `gitDirectory.appendingPathComponent("HEAD")`
  works in both cases.
- **Cgit2 import drops.** After each migration commit, `grep -n
  "git_\|Cgit2" <file>` should be your decider: if the file has no
  `git_*` references and no `Cgit2` references outside the to-be-
  deleted import, drop the import. Otherwise leave it.
- **`@testable import Git2` â†’ `import Git2`.** `Repository.create`,
  `Repository.open`, `Repository.isBare`, `Repository.isHeadUnborn`,
  `Repository.workingDirectory`, `Repository.gitDirectory`, `GitError`,
  `GitError.Code` are all `public`. A file that doesn't touch
  `internal` symbols (repository.lock, internal initializers, etc.)
  can drop `@testable`.
- **Line numbers.** Line numbers in this plan reference the
  pre-migration state of `main` at commit `2e7d936`. They shift as
  each task lands. Use the surrounding code (function name, test
  name) to locate the block when line numbers drift.
- **Single focused slice.** Don't add features, refactor, or clean
  up unrelated code while implementing this. Resist the urge to
  modernize surrounding tests or add `async` variants. Keep diffs
  minimal.
